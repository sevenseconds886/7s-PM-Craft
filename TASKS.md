## 五、TAPD MCP 任务拆解（T049）

### 需求概述
- **同步方向**：PM-Craft → TAPD（单向）
- **同步粒度**：单个需求手动发布
- **TAPD 目标**：不创建 Story，直接放入指定迭代（作为迭代需求）
- **附件**：原型 HTML 文件上传为附件
- **认证**：OAuth 2.0

### 归档说明（2026-05-28）

经评估，TAPD 发布功能暂时不需要内置到 PM-Craft 中。当前工作流下，通过 Agent 直接操作 TAPD API 发布需求更为灵活。

**已完成代码**：全部 8 个子任务开发完成，OAuth + API Token 双认证方案已验证通过（迭代列表 API 调用 200 OK）。

**归档位置**：`archive/tapd/`
- `tapd.js` — TAPD API 封装模块
- `server.js.bak.tapd` — 后端路由片段
- `index.html.bak.tapd` — 前端 UI 片段
- `app.js.bak.tapd` — 前端逻辑片段
- `README.md` — 恢复指南

**恢复方法**：见 `archive/tapd/README.md`

### 技术要点
- **TAPD API**：使用 `https://api.tapd.cn` 或企业版地址
- **OAuth 流程**：授权码模式，需注册 TAPD 应用获取 client_id/client_secret
- **Token 存储**：存储在 `.settings.json` 或独立文件，加密处理
- **状态映射**：PM-Craft 状态 ↔ TAPD 状态需可配置
- **优先级映射**：PM-Craft 优先级 ↔ TAPD 优先级需可配置
