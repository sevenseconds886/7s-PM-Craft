# 7s-PM-Craft 产品需求文档 v2.3

> 版本：v2.3 | 最后更新：2026-05-12 | Playwright测试：75/75 通过

---

## 1. 产品概述

### 1.1 产品定位

7s-PM-Craft 是一个**自托管的 Web 端需求管理平台**，采用**文件协议替代数据库**，所有数据以本地文件形式存储。面向产品经理和研发团队，提供从需求草稿收集、正式需求管理、原型展示到迭代规划的全流程支持。

### 1.2 核心价值

- **零配置**：npm install → npm start 即可运行，不依赖外部数据库
- **文件即数据**：所有需求以 Markdown + YAML 文件存储，Git 友好
- **AI 就绪**：标准化文件协议，可被 AI 工具（Skills）读写
- **全链路覆盖**：从灵感收集（需求池）→ 需求管理 → 原型展示 → 迭代规划

### 1.3 技术栈

| 层级 | 技术 |
|------|------|
| 后端 | Node.js + Express |
| 前端 | 原生 HTML/JS + Tailwind CSS (CDN) |
| 渲染 | marked.js (Markdown 渲染) |
| 测试 | Playwright E2E (75 例) |
| 存储 | 本地文件系统 |

#### 3.5.3 草稿状态转换

```
draft ──→ in_progress ──→ published ──→ archived
  ↑        ↑              │              │
  └────────┘              │              │
  (archived 可恢复为 draft)              │
                                         │
                    published 只能归档，不能回退
```

**状态转换规则**：
- `draft` → `in_progress` / `archived`
- `in_progress` → `draft` / `archived`
- `published` → `archived`（仅能归档）
- `archived` → `draft`（恢复为草稿）
- ❌ 禁止直接通过状态更新切换到 `published`（必须走发布流程）

#### 3.5.4 数据依赖

- `GET /api/drafts` — 获取草稿列表
- `POST /api/drafts` — 创建草稿
- `GET /api/drafts/:id` — 获取单个草稿
- `PUT /api/drafts/:id` — 更新草稿
- `POST /api/drafts/:id/status` — 更新草稿状态
- `POST /api/drafts/:id/publish` — 发布草稿为正式需求
- `DELETE /api/drafts/:id` — 删除草稿

## 7. 文件存储协议

### 7.1 目录结构

```
{workspace}/
├── products/
│   └── {product-line}/
│       └── REQ-{id}-{slug}/
│           ├── requirement.md
│           ├── prototype-web.html      # 可选
│           └── prototype-mobile.html   # 可选
├── archive/
│   └── {product-line}/
│       └── REQ-{id}-{slug}/
│           ├── requirement.md
│           ├── prototype-web.html
│           └── prototype-mobile.html
├── drafts/                              # 需求池
│   └── DRAFT-{id}-{slug}/
│       ├── draft.md                     # 主体文件
│       ├── requirement.md               # 别名（可选）
│       └── *.html                       # 原型文件（可选）
│   └── DRAFT-{id}-{slug}.md             # 旧格式（向后兼容）
├── .sprints.json                        # 迭代数据
├── .settings.json                       # 设置数据
└── skills/pm-craft-rules/SKILL.md       # 规范文件
```

### 7.2 ID 分配规则

- **需求 ID**：`REQ-` + 6位数字，全局自增
  - 扫描 `products/` 和 `archive/` 下所有 `REQ-*` 文件夹
  - 取最大编号 + 1

- **草稿 ID**：`DRAFT-` + 6位数字，自增
  - 扫描 `drafts/` 下所有草稿
  - 取最大编号 + 1

### 7.3 多产品线存储规则

- 物理文件夹路径由 `product_line` 数组**第一个元素**决定
- 支持一个需求关联多个产品线（元数据层面）
- 发布草稿到多个产品线时，每个产品线创建独立的需求副本
